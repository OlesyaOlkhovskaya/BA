Диаграмма последовательности: Создание и публикация курса @startuml actor Преподаватель participant "Веб-приложение" as ВебПриложение participant "API" as API participant "База данных" as БД participant "Сервис хранения файлов" as ФайловыйСервис Преподаватель -> ВебПриложение: Открыть страницу создания курса ВебПриложение -> Преподаватель: Отобразить форму создания курса Преподаватель -> ВебПриложение: Отправить данные курса (название, описание, файлы) ВебПриложение -> API: POST /courses (данные курса) API -> БД: INSERT INTO courses (данные курса) БД --> API: Подтверждение успешного сохранения API -> ФайловыйСервис: Сохранить файлы курса ФайловыйСервис --> API: Подтверждение успешного сохранения файлов API --> ВебПриложение: Успешное создание курса ВебПриложение -> Преподаватель: Уведомление о успешном создании курса Преподаватель -> ВебПриложение: Открыть страницу публикации курса ВебПриложение -> Преподаватель: Отобразить форму публикации курса Преподаватель -> ВебПриложение: Отправить запрос на публикацию курса ВебПриложение -> API: POST /courses/{courseId}/publish API -> БД: UPDATE courses SET status='published' WHERE id={courseId} БД --> API: Подтверждение успешного обновления API --> ВебПриложение: Успешная публикация курса ВебПриложение -> Преподаватель: Уведомление о успешной публикации курса @enduml
